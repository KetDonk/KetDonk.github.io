Arnica CRM

A developer-friendly, customizable CRM solution with dual-mode interface

View on GitHub Live Demo

Overview

Arnica CRM is a lightweight, highly customizable CRM system designed primarily for freelance programmers while remaining accessible to all users. Its unique dual-mode interface allows both developer-centric and standard user experiences, making it versatile for different user preferences and technical abilities.

Tech Stack

Frontend

TypeScript
React
Tailwind CSS

Backend

Node.js
Express.js
TypeScript

Key Features

Dual-Mode Interface

Switch between developer and standard modes to access different levels of functionality and complexity.

SQL Query Integration

Advanced filtering with SQL-like queries for developers, alongside a visual block-based filter builder for standard users.

Integrated Web Terminal

Execute commands and scripts directly through a web terminal interface with custom shortcuts.

Smart Contact Management

Create, manage, and organize contacts with custom fields, automated tagging, and duplicate detection.

Company Organization

Organize contacts under companies with isolated contact lists and customizable relationship management.

REST API & Integrations

Full REST API access with Python wrapper and integration hub for GitHub, GitLab, and more.

Development Roadmap

Base Development

  • Authentication System
  • RBAC Implementation
  • Database Schema

Core Features

  • Contact Management
  • Data Imports
  • Task Management

Advanced Features

  • SQL Query System
  • Web Terminal
  • API Integration

Sub-Projects

API TypeScript Wrapper

Official TypeScript wrapper for the Arnica API.

SQL Interpreter

Custom SQL-like query language interpreter for advanced filtering.

Desktop Client

Native desktop application for enhanced performance.

Documentation

Comprehensive documentation and guides.

CLI Tool

Command-line interface for automation and scripting.

Integrations Hub

Central hub for managing third-party integrations.

Development Standards

  • 80-character line width limit
  • Feature-based code organization
  • Emphasis on code readability over performance
  • Small, reusable components
  • Self-documenting code with meaningful comments
  • Comprehensive testing and CI/CD through GitHub Actions
